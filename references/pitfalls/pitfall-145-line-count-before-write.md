# Phase 3 文档行数超标（先写后删）

## 问题

Phase 3 写 `constitution.md` 和 `conversation.md` 时没先检查行数限制（constitution ≤ 60 行，conversation ≤ 60 行），写完后发现超限（66 行、71 行），不得不重写。

## 根因

行数限制记录在 Phase 3 workflow 文件和 SKILL.md 中，但写作时没有主动应用。灵犀倾向于先写内容再检查格式，而不是写作时就控制篇幅。

## 修复方法

**写作前先计算预算：**

```
constitution.md ≤ 60 行
├── frontmatter: ~5 行
├── 标题/概述: ~3 行
├── 代码规范: ~8 行
├── 架构约束: ~8 行
├── 禁止: ~10 行
├── 验收标准: ~8 行
└── 实现细节规范: ~18 行（压缩为 6 个子节，每节 3 行）
```

**写作时：**
1. 先列出要写的章节，估算每章行数
2. 总和超限时，压缩最长的章节（合并条目、减少描述）
3. 写完立即 `wc -l` 验证
4. 超标 → 删减后再写入，不要写完再删

**conversation.md ≤ 60 行的压缩技巧：**
- 合并"需求概述"和"调研发现"为一节
- 技术选型用表格（紧凑）而不是分段描述
- 质量规格用表格而不是列表
- 决策用 bullet list 而不是分段

## 反例

先写 70+ 行的完整版 → `wc -l` 发现超限 → 重写精简版。浪费一轮工具调用。
