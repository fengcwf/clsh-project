# 技能库查询优先级

## 铁律：查 skill 先于问用户

**当遇到配置、凭据、环境参数等问题时，必须先查 skill 库，禁止直接问用户。**

### 查询优先级

1. **查 skill 库** — 相关 skill 的 references/、SKILL.md
2. **查 memory** — 之前记录的环境信息
3. **查 wiki** — Obsidian 项目文档
4. **问用户** — 最后手段

### 典型场景

| 场景 | 应该查的 skill | 问用户的后果 |
|------|---------------|-------------|
| 网络连接失败 | `proxy-workarounds` | 用户反感（"为什么需要问我"） |
| Git 推送失败 | `proxy-workarounds` | 同上 |
| API 调用失败 | 相关 API skill | 浪费时间 |
| 部署配置 | 相关部署 skill | 可能遗漏关键参数 |

### 自检清单

在问用户配置/凭据/环境参数之前，必须确认：
- [ ] 已搜索 skills_list
- [ ] 已查看相关 skill 的 SKILL.md
- [ ] 已查看相关 skill 的 references/
- [ ] 已查 memory
- [ ] 以上都找不到，才问用户
