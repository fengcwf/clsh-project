# Phase 确认模板

每个 Phase 结束时，使用以下模板输出确认请求。`[CODE]` 占位符**必须**替换为实际生成的确认码。

> **注意：** 模板已内嵌到 SKILL.md 各 Phase 节中。此文件为备份参考，主流程以 SKILL.md 内嵌模板为准。

---

```
需求澄清完成。

- [决策1]: [结果]
- [决策2]: [结果]
...

---

🔑 `[CODE]`
📋 复制上面的码回复即可
---
```

## 使用规则

1. **[CODE] 是占位符** — LLM 填充时**必须**调用 `python3 -c "import secrets,string; print(''.join(secrets.choice(string.ascii_uppercase+string.digits) for _ in range(6)))"` 生成实际码
2. **不生成码 = 流程违规** — 模板中没有实际码就输出 = 漏了
3. **码不匹配不放行** — 大佬回复的码必须与生成的码完全匹配
4. **码只用一次** — 每个决策点生成新码，不复用
5. **格式必须一致** — `` 🔑 `[CODE]` `` + `📋 复制上面的码回复即可`，两件套缺一不可
6. **回复格式** — 大佬直接回复码即可（不需要"确认"前缀），方便复制
