**规则：**
- Wave 0 的 tester 卡数 ≤ 3
- 每个 tester 卡可包含多个子步骤（验证同一个功能点的多个方面）
- 子步骤必须是**可执行的**（命令、浏览器操作、文件检查），不能是"看看对不对"
- tester 必须实际运行命令/读代码，不能只看文件名
- Wave 0 通过后才能进入 Wave 1

## 使用规则

1. **每个 `[CODE]` 占位符** — 填充时必须 `python3 -c "import secrets,string; print(''.join(secrets.choice(string.ascii_uppercase+string.digits) for _ in range(6)))"` 生成
2. **Wave 0 不可跳过** — 没有 Wave 0 tester 验证就派 coder = 流程违规
3. **顺序派发不可违反** — 同时派发 2+ 个 coder 任务 = 流程违规
4. **每个 `[ ]` 打勾 `[x]`** — 完成后必须标记，未标记 = 未完成
5. **中间验证不可跳过** — 每 2-3 个 coder 任务必须暂停做 tester 验证
6. **多步 Task 必须属于同一 Vertical Slice** — 跨功能 = 拆分
7. **Tester 子步骤必须可执行** — "看看对不对" ≠ 子步骤
8. **确认码格式统一** — `🔑` + 反引号 + `📋` 三件套
